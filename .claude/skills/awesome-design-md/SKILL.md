# Awesome DESIGN.md — Reference Design Systems

A library of analyzed, production-grade design systems stored as `DESIGN.md`
files in `references/`. Each is a plain-markdown spec with concrete tokens
(colors, typography, spacing, radii, shadows, motion) and usage rules that an
agent can read and apply directly.

## When to use
Use this whenever building, styling, or redesigning UI and you want the result
to look intentional and consistent rather than templated. Pick ONE primary
reference that matches the desired feel, optionally borrow details from a
second, then translate the tokens into the project's own brand.

## How to use
1. Decide the desired feeling (trustworthy, editorial, premium, technical, …).
2. Open the matching folder in `references/<brand>/DESIGN.md` and read its
   tokens and rules.
3. Adapt — do not copy verbatim. Re-map colors/fonts to the project brand and
   keep the structural decisions (scale ratios, spacing rhythm, motion feel).
4. Record the chosen direction in the project's own design tokens so every
   section stays consistent.

## Recommended references for a law-firm / professional-services site
Serious, trustworthy, editorial — avoid loud consumer/gaming aesthetics.

- **stripe** — financial-trust polish, restrained palette, editorial headlines.
- **linear.app** — precise typographic rhythm, calm dark/light, crisp motion.
- **notion** — warm, readable, content-first, approachable professionalism.
- **sanity** / **mintlify** — clean editorial documentation feel, strong type.
- **wise** / **revolut** — trustworthy fintech clarity, confident color use.
- **superhuman** / **vercel** / **apple** — premium minimal, generous spacing.

Avoid for this project: nintendo-2001, playstation, ferrari, lamborghini,
nike, starbucks (too brand-loud / off-tone for legal services) — unless a
specific accent is wanted deliberately.
